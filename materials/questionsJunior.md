### Общие вопросы

1. Основные принципы ООП
<details> <summary>Ответ</summary> <p>
    1. Инкапсуляция - получение данных через публичные методы<br>
    2. Наследование - создание новых классов на основе существующих<br>
    3. Полиморфизм - использование одинакового интерфейса<br>
    4. Абстракция - позволяет выделить общие характеристики объекта<br>
</p> </details>

3. Стек и очередь
<details>
    <summary>Ответ</summary>
        <p>
        1. Стек - абстрактный тип данных, представляющий собой список элементов, организованных по принципу LIFO (англ. last in — first out, «последним пришёл — первым вышел»)<br>
        2. Очередь - последний пришел последний вышел<br>
        </p>
</details>

4. Что такое шаблонный класс или функци
<details>
    <summary>Ответ</summary>
        <p>
            1. Обобщенный класс позволяет работать с разными типами данных ключевое слово template<br>
            <pre>
	            <code>
                template <typename T>  // Объявление шаблонного параметра
                class Wrapper {
                    private:
                        T value;          // Поле типа T
                    public:
                        Wrapper(T val) : value(val) {}  // Конструктор
                        T getValue() const { return value; }  // Метод доступа
                        void setValue(T val) { value = val; } // Метод изменения
                };
	            </code>
            </pre>
            2. Шаблонная функция
            <pre>
	            <code>
                template<typename Type>
                Type max(Type a, Type b){ 
                    return (a >= b ? a : b);
                }
	            </code>
            </pre>
        </p>
</details>

6. Может ли быть виртуальная функция шаблонной
<details>
    <summary>Ответ</summary>
        <p>
        Виртуальная функция это функция котороую можно переопределить в наследуемом классе, чтобы сохранить ее сигнатуру
        Можно оценить ее как динамический полиморфизм, а temlate как статический полиморфизм <br>
        Отвечая на вопрос виртуальная функция шаблонной быть не может, так как шаблонная функция не является функцией т.к. она генерируется на этапе компиляции
        </p>
</details>

9. Что такое препроцессор?
<details>
    <summary>Ответ</summary>
        <p>
        Препроцесор - парсер который запускается до компиляции программы убирает коментарии, включает include, вставляет define
        </p>
</details>

10. Какие знаете команды препроцессора?
<details>
    <summary>Ответ</summary>
        <p>
        #include — вставляет код из указанного файла в текущий файл.<br>
        #define — создаёт символическую или макроконстанту.<br>
        #undef — удаляет макрос, который уже был определён.<br>
        #if, #elif, #else, #endif — условная компиляция на основе выражения.<br>
        #ifdef, #ifndef — условная компиляция на наличие или отсутствие макроса.<br>
        #error — останавливает процесс компиляции и выводит сообщение об ошибке.<br>
        #pragma — предоставляет инструкции компилятору, зависящие от реализации.<br>
        </p>
</details>
